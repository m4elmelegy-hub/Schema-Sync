Build a complete professional Financial Lock System for the ERP.

⚠️ GOAL:
Protect closed financial periods from direct modification while still allowing safe accounting corrections through reversal/adjustment entries and full audit trail.

This must be implemented as an accounting-grade control system, not a simple UI lock.

==================================================

1. CORE CONCEPT
   ==================================================

After a period is financially locked:

❌ No direct edit of old records
❌ No direct delete of old records
❌ No changing amounts/dates/customers/safes/invoices inside locked periods

✅ Corrections must happen only through:

* reversal entries
* adjustment entries
* return transactions
* compensating vouchers

This is the critical accounting rule.

======================

When a record date is <= locked_until_date:

Prevent direct modification of:

* receipt vouchers
* payment vouchers
* treasury actions
* expenses
* sales invoices
* purchase invoices
* customer balances via old edits
* supplier balances via old edits
* journal entries (if appropriate)

Protected operations:

* edit
* delete
* change date
* repost/undo in locked period

7) CORRECTION SYSTEM (CRITICAL)
===============================

Do NOT allow old record edit after lock.

Instead implement safe correction workflows.

A) Reverse Voucher
For locked receipt/payment:

* create opposite voucher
* same amount
* linked to original voucher
* marked as "عكسي"

B) Adjustment Voucher
Allow new correction entry with:

* correction reason
* linked original record
* created_by
* created_at

C) Sales correction
Use:

* return invoice
* correction invoice
  NOT direct invoice mutation

D) Expense correction
Use:

* negative reversing expense
* then correct new entry
