Build a full professional Arabic ERP system (POS + Accounting + Inventory) using:

Frontend:

* HTML, CSS (modern dark glass UI, Arabic RTL)
* Vanilla JavaScript

Backend:

* Supabase (PostgreSQL)

---

## CORE SYSTEM REQUIREMENTS

1. Authentication:

* Login / Signup
* Roles: admin / employee

---

2. CUSTOMERS & SUPPLIERS (NO NEED TO PRE-REGISTER)

---

* System must allow:

  * Cash transactions (no name)
  * Credit transactions (enter name manually OR select existing)

Tables:

* customers (optional)
* suppliers (optional)

Fields:

* name
* phone
* balance (auto calculated)

---

3. SALES SYSTEM (VERY IMPORTANT)

---

Support 3 types:

1. Cash sale
2. Credit sale (آجل)
3. Partial payment

Fields:

* invoice_no
* customer_name
* payment_type (cash / credit)
* total_amount
* paid_amount
* remaining_amount
* status (paid / partial / unpaid)

Rules:

* If paid_amount < total → remaining goes to customer balance
* If credit → full amount added to customer balance

---

4. PURCHASE SYSTEM

---

Same logic as sales:

* Cash purchase
* Credit purchase from supplier
* Partial payment

Fields:

* supplier_name
* payment_type
* total_amount
* paid_amount
* remaining_amount

Rules:

* Remaining goes as debt to supplier

---

5. PRODUCTS

---

Fields:

* name
* sku
* category
* quantity
* cost_price
* sale_price
* low_stock_threshold

---

6. DAILY ACCOUNTING (IMPORTANT)

Create unified "transactions" table:

---

* When paying supplier
