🔒 مهم جداً - طلب تقرير شامل لمراجعة الأمان (Security Audit)

أحتاج إعداد تقرير تقني كامل لمراجعة أمان النظام قبل النشر، يرجى استخراج وتوفير المعلومات التالية:

📁 **1. هيكل المشروع والتقنيات (Tech Stack)**
- قائمة بجميع الملفات الرئيسية (Tree Structure)
- لغة البرمجة والإطار المستخدم (Framework)
- قاعدة البيانات ونوعها (PostgreSQL/MySQL/MongoDB...)
- المكتبات الخارجية المستخدمة (package.json/requirements.txt)

🔐 **2. نظام المصادقة والصلاحيات (Auth System)**
- ملفات middleware للمصادقة
- كود تسجيل الدخول (Login Controller)
- كود التسجيل (Register Controller)
- آلية إنشاء و التحقق من JWT/Session
- نظام الصلاحيات (RBAC) - كيف يتم التحقق من صلاحيات المستخدم؟

🗄️ **3. قاعدة البيانات والاستعلامات**
- Schema كامل للـ Database (Tables/Models)
- جميع دوال الـ CRUD التي تتعامل مع بيانات حساسة (المالية، المستخدمين، الموارد البشرية)
- هل يستخدم ORM أم SQL خام؟ (أرسل أمثلة على الاستعلامات)

🌐 **4. نقاط النهاية (API Endpoints)**
- قائمة كاملة بجميع Routes/Endpoints خاصة:
  * تسجيل الدخول والخروج
  * إدارة المستخدمين
  * العمليات المالية (إنشاء/تعديل/حذف الفواتير)
  * رفع/تحميل الملفات
- هل توجد Rate Limiting أو حماية من Brute Force؟

📤 **5. إدارة الملفات (File Upload)**
- كود رفع الملفات (إن وجد)
- أنواع الملفات المسموحة والتحقق منها
- مكان تخزين الملفات (هل في public أم private storage؟)

🔑 **6. المتغيرات البيئية (Environment Variables)**
- قائمة بجميع Variables في .env file (أرسل الأسماء فقط بدون القيم الحساسة)
- هل هناك مفاتيح API أو كلمات مرور مكتوبة hardcoded في الكود؟

📊 **7. السجلات والتدقيق (Logs & Audit)**
- هل يتم تسجيل العمليات الحساسة (من أنشأ/عدل/حذف)؟
- نظام الـ Logging المستخدم

⚠️ **8. معالجة الأخطاء (Error Handling)**
- كيف يتم إرجاع الأخطاء للمستخدم؟ (هل يظهر SQL errors مباشرة؟)
- هل توجد صفحات 404/500 مخصصة؟

📦 **9. إعدادات الأمان الأساسية**
- هل يوجد CORS configuration؟ ما المواقع المسموح لها؟
- Security Headers (X-Frame-Options, CSP, etc.)
- هل يتم تطهير المدخلات (Input Sanitization) في الحقول؟

🔍 **10. اختبار سريع**
- هل يمكنك فحص:
  * هل توجد ثغرات SQL Injection في أي form؟
  * هل توجد ثغرات XSS في عرض البيانات؟
  * هل يمكن الوصول لبيانات المستخدمين الآخرين بتغيير ID في URL (IDOR)؟

**من فضلك قدم المخرجات على شكل:**
1. تقرير نصي مفصل
2. لصق الكود المهم في Code Blocks منفصلة
3. تحديد الملفات التي تحتاج انتباه خاص (Security Hotspots)
