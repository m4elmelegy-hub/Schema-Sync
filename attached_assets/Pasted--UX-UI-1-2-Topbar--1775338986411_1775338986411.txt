أحتاج تنفيذ تحسينات UX/UI على صفحتين:

1. صفحة إضافة مستخدم (الصلاحيات)
2. Topbar (الإشعارات + المستخدم + الثيم)

⚠️ مهم:
لا تغيّر أي business logic أو backend — فقط UI/UX

========================
🔵 أولاً: صفحة الصلاحيات (Permissions Page)
===========================================

المطلوب تنفيذ تصميم Hybrid حديث:

1. Layout عام:

* الصفحة تكون compact (غير طويلة)
* استخدام sections مقسمة (المبيعات / المخزون / المالية...)
* كل section يحتوي على:

  * عنوان
  * badge (مثلاً: 2 / 6)

2. داخل كل section:

* عرض الصلاحيات في Grid (عمودين)
* كل صلاحية عبارة عن row:

  * اسم الصلاحية (يمين)
  * Toggle (يسار)

3. Toggle System:

* نفس فكرة زرار dark/light mode
* ON:
  لون accent (orange أو green)
* OFF:
  رمادي
* animation خفيفة عند التغيير

4. إزالة:

* checkboxes
* pills القديمة
* أي تصميم قديم للصلاحيات

5. Search:

* input أعلى الصفحة
* فلترة الصلاحيات أثناء الكتابة

6. UX إضافي:

* hover على الصف → highlight خفيف
* عند تفعيل toggle → الخلفية تتغير بشكل subtle

7. Buttons:

* زر حفظ + إلغاء
* ثابتين أسفل الصفحة (sticky)

========================
🟣 ثانياً: Topbar تحسين كامل
============================

المشكلة الحالية:

* الإشعارات صغيرة
* اسم المستخدم اختفى
* العناصر غير متوازنة

المطلوب:

1. ترتيب العناصر (RTL):

[ عنوان الصفحة ]   [ Search ]   [ 🔔 ]   [ 🌙 ]   [ Avatar + Name ]

2. Notifications:

* تكبير حجم زر الإشعارات
* badge واضح (عدد التنبيهات)
* padding مناسب

3. User Info:

* إعادة:

  * اسم المستخدم
  * الدور (Admin / User)
* بجانب avatar

4. Theme Toggle:

* نفس التصميم الحالي
* لكن محاذاة صحيحة

5. Spacing:

* gap ثابت بين العناصر (16px أو 20px)
* alignment رأسي مضبوط

6. Search:

* واضح ومناسب
* نفس height لباقي العناصر

========================
🎯 الهدف النهائي
================

* صفحة صلاحيات:
  سهلة + واضحة + سريعة

* Topbar:
  متوازن + احترافي + واضح

* النظام كله:
  SaaS-level UI
  بدون تعقيد أو زحمة

========================
📦 Output المطلوب
=================

* تحديث components الحالية (مش إنشاء جديدة)
* الحفاظ على نفس theme system
* كود reusable
