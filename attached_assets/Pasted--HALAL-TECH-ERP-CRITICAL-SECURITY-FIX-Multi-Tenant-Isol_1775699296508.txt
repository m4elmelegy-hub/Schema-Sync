# [HALAL TECH ERP — CRITICAL SECURITY FIX: Multi-Tenant Isolation]

## ⚠️ CRITICAL ISSUE
The system has a severe security vulnerability: ALL users from ANY company can see and DELETE data belonging to OTHER companies. This is because:
1. JWT token does NOT include `company_id`
2. NO queries filter by `company_id`
3. There is NO tenant middleware

This must be fixed completely and without breaking existing functionality.

---

## Fix 1: Update JWT to include company_id

In `artifacts/api-server/src/middleware/auth.ts`:

Change `signToken` to include company_id:
```typescript
// BEFORE
export function signToken(userId: number, role: string): string {
  return jwt.sign({ userId, role }, JWT_SECRET, { expiresIn: "12h" });
}

// AFTER
export function signToken(userId: number, role: string, companyId: number): string {
  return jwt.sign({ userId, role, companyId }, JWT_SECRET, { expiresIn: "12h" });
}
```

Update the `authenticate` middleware to extract and attach `companyId` from JWT:
```typescript
// After verifying token, add:
req.companyId = decoded.companyId;
req.userId = decoded.userId;
req.role = decoded.role;
```

Add to Express Request type declaration:
```typescript
declare global {
  namespace Express {
    interface Request {
      userId: number;
      role: string;
      companyId: number;
    }
  }
}
```
