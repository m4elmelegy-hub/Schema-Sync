# [HALAL TECH ERP — إدارة المديرين العامين]

## المطلوب

إضافة تبويب جديد في لوحة تحكم المدير العام لإدارة حسابات المديرين العامين (super_admin).

-----

## 1. إضافة تبويب “المديرون العامون” في الصفحة

في `artifacts/erp-system/src/pages/super-admin.tsx`:

أضف تبويبين (tabs) في أعلى الصفحة:

```
[ 🏢 الشركات المسجلة ]  [ 👑 المديرون العامون ]
```

- التبويب النشط له خلفية برتقالية
- التبويب الغير نشط outlined داكن
- المحتوى الحالي (الشركات) يبقى تحت التبويب الأول

-----

## 2. محتوى تبويب “المديرون العامون”

### جدول المديرين:

|عمود|المحتوى|
|----|-------|

# | رقم تسلسلي

الاسم | name من erp_users
اسم المستخدم | username
آخر دخول | last_login (أو “لم يسجل بعد”)
الحالة | active (نشط/موقوف)
الإجراءات | أزرار التعديل والإيقاف والحذف

### تصميم الجدول:

- نفس تصميم جدول الشركات (داكن + برتقالي)
- صف الهيدر برتقالي
- صفوف متناوبة الألوان

-----

## 3. إضافة مدير عام جديد

زرار “➕ مدير عام جديد” بلون برتقالي يفتح modal يحتوي على:

```
- الاسم الكامل (إلزامي)
- اسم المستخدم (إلزامي - بدون مسافات)
- الرقم السري (إلزامي - 4 أحرف على الأقل)
- تأكيد الرقم السري (إلزامي - يطابق الأول)
```

Validations:

- اسم المستخدم لا يحتوي على مسافات
- اسم المستخدم غير مكرر
- الرقم السري لا يقل عن 4 أحرف
- تأكيد الرقم السري يطابق الأول

بعد الإضافة: toast نجاح + تحديث القائمة

-----

## 4. تعديل بيانات مدير عام

زرار ✏️ بجانب كل مدير يفتح modal تعديل:

```
- الاسم الكامل
- اسم المستخدم
- الرقم السري الجديد (اختياري - اتركه فارغاً إذا لم تريد تغييره)
- تأكيد الرقم السري الجديد
```

-----

## 5. إيقاف/تفعيل مدير عام

زرار 🔴/🟢 يغير حالة active:

- مدير نشط → زرار “إيقاف” أحمر
- مدير موقوف → زرار “تفعيل” أخضر
- لا يمكن إيقاف المدير الذي قام بتسجيل الدخول حالياً
- رسالة خطأ: “لا يمكن إيقاف حسابك الحالي”

-----

## 6. حذف مدير عام

زرار 🗑️ أحمر مع confirmation dialog:

```
"هل أنت متأكد من حذف المدير [الاسم]؟"
"لا يمكن التراجع عن هذا الإجراء."
زر: "نعم، احذف" (أحمر) | "إلغاء" (رمادي)
```

- لا يمكن حذف المدير الذي قام بتسجيل الدخول حالياً
- يجب أن يبقى مدير عام واحد على الأقل في النظام
- رسالة خطأ: “يجب أن يكون هناك مدير عام واحد على الأقل”

-----

## 7. Backend API endpoints جديدة

في `artifacts/api-server/src/routes/super-admin.ts` أضف:

```
GET    /api/super-admin/managers         ← قائمة كل super_admin
POST   /api/super-admin/managers         ← إضافة مدير جديد
PATCH  /api/super-admin/managers/:id     ← تعديل بيانات مدير
DELETE /api/super-admin/managers/:id     ← حذف مدير
```

Rules في الـ backend:

- كل هذه الـ endpoints تتطلب role = super_admin
- POST: hash الـ PIN بـ bcrypt قبل الحفظ
- PATCH: لو PIN جديد موجود → hash قبل الحفظ، لو فارغ → لا تغير الـ PIN
- DELETE: تحقق إن المديرين العامين أكثر من 1 قبل الحذف
- DELETE: لا تسمح بحذف نفس المستخدم المسجل دخوله

-----

## 8. الأمان

- كل الـ endpoints تتحقق من JWT وأن role = super_admin
- لا يمكن ترقية أي مستخدم عادي إلى super_admin من هنا (role ثابت = super_admin عند الإنشاء)
- PIN يُخزن مشفراً دائماً
