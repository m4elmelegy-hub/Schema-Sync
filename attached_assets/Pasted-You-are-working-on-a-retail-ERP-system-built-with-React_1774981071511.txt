Create POST /api/inventory/opening-balance

Body: { product_id, quantity, cost_price, date, notes }

Logic:
1. Check if opening balance already exists for this product
   → If yes, BLOCK with error: "رصيد أول المدة مسجل مسبقاً"
2. Update products.quantity += quantity
3. Recalculate weighted avg cost:
   new_cost = (old_qty × old_cost + qty × cost_price) / (old_qty + qty)
   Update products.cost_price = new_cost
4. INSERT into stock_movements:
   movement_type = 'opening_balance'
   quantity_before = old_qty
   quantity_after = old_qty + quantity
   unit_cost = cost_price
   reference_type = 'opening_balance'
   notes = notes
5. Also create POST /api/opening-balance/treasury:
   Body: { safe_id, amount, date, notes }
   → Update safes.balance += amount
   → INSERT into transactions with type='opening_balance', direction='in'
6. Also create POST /api/opening-balance/customer:
   Body: { customer_id, amount, date, notes }
   → Update customers.balance += amount
7. Also create POST /api/opening-balance/supplier:
   Body: { supplier_id, amount, date, notes }
   → Update suppliers.balance += amount

Build an Opening Balance screen in Settings (admin only):
- Tab 1: Treasury opening balances (per safe)

- Tab 3: Customer opening balances
