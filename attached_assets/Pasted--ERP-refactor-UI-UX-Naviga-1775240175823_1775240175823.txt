راجع واجهة مشروع ERP بالكامل قبل أي تعديل، ولا تنفذ refactor واسع.

المطلوب الآن هو UI/UX + Navigation + Role-based visibility audit فقط، ثم خطة تنفيذ تدريجية آمنة.

قواعد مهمة:
- لا تعدل أي كود الآن
- لا تعمل refactor كبير
- لا تغير business logic إلا إذا كان مطلوبًا صراحة
- اعمل review أولًا
- لو وجدت مشكلة متكررة، اجمعها في pattern واحد
- أخرج الخطة على مراحل صغيرة يمكن تنفيذها خطوة بخطوة

راجع هذه المحاور:

1) Design System / UI Consistency
- مشاكل الألوان
- مشاكل الوضع الليلي والنهاري
- التباين والقراءة
- توحيد البطاقات والجداول والفلاتر والعناوين
- الـ spacing والهوامش والأحجام
- الـ z-index خصوصًا الإشعارات/التوست/المودال/الدروب داون
- هل الإشعارات تظهر خلف الكروت أو أي عنصر أعلى منها

2) Information Architecture / Navigation
- راجع تكرار الصفحات
- راجع وجود صفحة Profits مستقلة وفي نفس الوقت تبويب أرباح داخل Reports
- اقترح هل تبقى مستقلة أم تُدمج أم تُبسط
- راجع الصفحات غير المفعلة أو غير المكتملة
- اقترح ما الذي يجب إخفاؤه مؤقتًا من الـ nav
- اقترح ما الذي يجب إعادة ترتيبه في الشريط الجانبي

3) Branch / Warehouse / Role UX
- كيف يرى admin كل الفروع وكل المخازن
- كيف يرى manager ما يخص الفرع/الفروع المسموح بها
- كيف يرى cashier أو salesperson فقط الخزينة/المخزن أو العمليات المسموح بها
- اقترح UX واضح لاختيار الفرع/المخزن عند الحاجة
- ميّز بين صلاحية الرؤية وصلاحية التعديل

4) Multi-company readiness from UI perspective
- راجع أين يجب أن يظهر company context
- راجع إن كانت الصفحات الحالية جاهزة لوجود أكثر من شركة
- لا تنفذ multi-tenant changes الآن، فقط حدد gaps المطلوبة

المخرجات المطلوبة:
A. قائمة مشاكل مرتبة بالأولوية:
- Critical
- High
- Medium
- Low

B. لكل مشكلة:
- اسم الصفحة/الصفحات
- وصف المشكلة
- سببها المحتمل
- التأثير على المستخدم
- أقل تعديل آمن ممكن

C. اقترح تنفيذ على 3 مراحل:
- Phase 1: UI bugs + z-index + dark/light + hide incomplete pages
- Phase 2: navigation cleanup + reports/profits decision + page consistency
- Phase 3: role-based visibility + branch/warehouse UX

D. لا تعدل أي ملف الآن
E. اعرض أسماء الملفات المتوقع تعديلها لاحقًا فقط
F. بعد الخطة انتظر الموافقة قبل التنفيذ
